## 2. 数据类型, 运算符和表达式
### 2.1 数据类型
1. 数据类型包括: 基本类型+构造类型+指针类型+空类型.
2. 基本类型:
    - 包括: 整型+浮点型+字符类型;
    - 所占字节数: 
        - 最小位数保证;
        - 1 == sizeof ( char ) ≤ sizeof ( short ) ≤ sizeof ( int ) ≤ sizeof ( long ) ≤ sizeof ( long long );
    - 存储区别:
        - 整型以补码存储(正数的补码是其本身, 负数的补码为其绝对值取反+1);
        - 浮点数存储, 以32位float类型举例: 1bit符号位+8bit指数位+23bit精度位;
        - 字符类型char, C语言把字符当作小整数进行处理;
    - 不同类型数据之间的转换:
        - 隐式转换: 整值提升+赋值转换;
        - 显式转换: 强制转换;
    - 特别注意:
        - char类型是由实现定义的, 取决与编译器(gcc通常默认为signed char);
        - 浮点类型的运算结果可能不准确, 无法直接比较;
        - bool类型(_Bool), 实际类型是unsigned char(gcc默认), 但是只能取值0或1;
        - 不同类型的0值: 0, '0', "0", '\0', NULL;
        - 输入输出函数中的占位符要与数据类型匹配;
3. 构造类型: 数组+结构体(struct)+共用体(union)+枚举(enum);
### 2.2 常量和变量
1. 常量:
    - 定义: 在程序执行过程中值不会发生变化的量;
    - 分类: 整型常量, 实型常量, 字符常量, 字符串常量, 标识常量;
        - 整型常量: 1, 7900, 0x7fff, 0b1111, 0657;
        - 实型常量: 1.0, 1.5e-2, 0x1.0p0, 0x1.0p-2;
        - 字符常量: 由单引号括起来的一个字符, 如'a', '1', ' ', '\0617', '\fe';
        - 字符串常量: 由双引号括起来的一个字符序列, 如"hello world", "1.0", "", "\0617", "\fe";
        - 标识常量: define宏定义, 无类型安全, 纯文本替换;
2. 变量:
    - 定义: 用来保存一些特定内容, 并且在程序执行过程中值随时发生变化的量;
    - 变量定义: [存储类型] 数据类型 标识符 = 值;
        - 标识符: 由字母、数字、下划线组成的单词, 不能以数字开头;
        - 数据类型: 基本数据类型+构造类型+指针类型;
        - 存储类型: static, auto, register(建议型), extern(说明型);
            - auto: 默认存储类型, 栈上存储, 自动分配和回收空间;
            - register: 寄存器存储类型, cpu寄存器存储, 只能定义局部变量, 大小固定(1个字长), 无地址;
            - static: 静态存储类型, 静态区存储, 生命周期贯穿整个程序运行期, 自动初始化为0值;
            - extern: 说明型, 不能改变被说明的变量的值或类型;
    - 作用域(scope), 生命周期(lifetime)和链接性(linkage):
        - 作用域:
            - 定义: 标识符在源代码中的可见范围, 由程序的词法结构决定;
            - 性质: 静态的, 编译器在编译时即可确定;
            - 分类: 块作用域, 文件作用域, 函数作用域, 函数原型作用域;
            - 作用: 决定标识符在哪里可以被直接使用;
        - 链接性:
            - 定义: 标识符(对象或函数)在其他作用域或翻译单元中被引用的能力;
            - 性质: 静态的, 由声明位置和存储类型决定, 在编译和链接时处理;
            - 分类: 无链接, 内部链接, 外部链接;
            - 作用: 控制多文件项目中标识符的共享和隐藏;
        - 生命周期:
            - 定义: 对象从创建到被销毁的整个过程时间段;
            - 性质: 动态的, 由程序运行时决定;
            - 分类: 静态存储期, 自动存储期, 动态存储期, 线程存储期;
            - 作用: 决定对象在运行时何时可用, 何时失效;
        - 关键点:
            - 作用域和链接性与标识符相关, 而生命周期与内存对象相关;
            - 作用域和链接性不一定对应;
            - 生命周期可以和作用域不一致;
    - 变量比较总结:
        | 变量类型 | 作用域 | 链接性 | 生命周期 | 存储位置 |
        |---------|--------|--------|----------|----------|
        | 全局变量 | 文件作用域 | 外部链接 | 静态存储期 | 静态区 |
        | 静态全局变量 | 文件作用域 | 内部链接 | 静态存储期 | 静态区 |
        | 局部变量 | 块作用域 | 无链接 | 自动存储期 | 栈 |
        | 静态局部变量 | 块作用域 | 无链接 | 静态存储期 | 静态区 |
        | 寄存器变量 | 块作用域 | 无链接 | 自动存储期 | CPU 寄存器 |
        | 动态分配变量 | 无固定作用域 | 无链接 | 动态存储期 | 堆 |
### 2.3 运算符和表达式
1. 运算符:
    - 分类: 
        - 算术运算符(+, -, *, /, %, ++, --);
        - 关系运算符(<, >, <=, >=, ==, !=);
        - 逻辑运算符(&&, ||, !);
        - 位运算符(&, |, ^, ~, <<, >>);
        - 赋值运算符(=, +=, -=, *=, /=, %=, <<=, >>=, &=, ^=, |=);
        - 条件运算符(?:);
        - 逗号运算符(,);
        - 指针运算符(*, &);
        - 分量运算符(., ->);
        - sizeof运算符(sizeof);
        - 类型转换运算符();
        - 下标运算符[];
        - 函数调用运算符();
    - 结合性和优先级;
    - 特殊用法:
        - 逻辑运算符(||, &&)的短路特性;
    - 位运算:
        - 将操作数第n位置1, 其他位不变: number |= (1 << n);
        - 将操作数第n位置0, 其他位不变: number &= ~(1 << n);
        - 测试第n位: number & (1 << n);
2. 表达式
    - 定义: 由运算符和操作数组成的序列，计算后会产生一个值，并且可能伴随副作用;
    - 性质: 是类型系统和值类别研究的核心, 表达式构成统一的计算模型;
    - 原因: 
        - 在汇编中，几乎每条指令(如 add、mov)都会产生结果或副作用, 类似于表达式;
        - C语言保留了这种“计算为核心”的思想，将大多数操作设计为表达式，使代码更接近机器思维，高效且简洁;
    - 代价:
        - 复杂的求值顺序规则: C/C++中很多表达式的字表达式求值顺序是未指定的(如函数参数, 运算符操作数), 往往由编译器决定, 带来可移植性的问题;
        - 副作用和序列点;
        - 值类别的复杂性;
## 3. 输入输出专题
### 3.1 格式化输入输出函数: printf和scanf
1. printf函数:
    - 函数原型: int printf(const char *restrict format, ...);
    - format参数转换说明格式: %[flags][width][.precision][length-specifier]specifier, 简化"%[修饰符]格式字符"
        - 引导符%;
        - 用于修改转换行为的标志flags:
            - -: 减号，左对齐;
            - +: 有符号数显示添加符号(正or负);
            - 空格: 符号数显示添加空格;
            - #: 添加进制前缀(八进制的0, 十六进制的0x或0X);
            - 0: 零填充;
        - 宽度width: 输出宽度, 默认填充空格;
        - 精度precision: 一般用于浮点数打印, 小数点后的位数, 默认为6;
        - 长度修饰符length-specifier: 主要用于short, unsigned short, long, long long等打印;
    - C语言编译器不会检测格式串中转换说明的数量是否和输出项的数量相匹配;
    - 带缓冲机制;
2. scanf函数:
    - 函数原型: int scanf(const char *restrict format, ...);
    - "模式匹配";
### 3.2 字符输入输出函数: getchar和putchar
1. getchar函数:
    - 函数原型: int getchar(void);
    - 返回值: 整型值, 输入的字符强转为整型值 or EOF;
2. putchar函数:
    - 函数原型: int putchar(int c);
    - 函数参数: 整型值, 要输出的字符;
    - 返回值: 整型值, 输出的字符强转为整型值 or EOF;
### 3.3 字符串输入输出函数: gets和puts
1. gets函数:
    - 函数原型: [[deprecated]] char *gets(char *s);
    - 缺陷: 不检查缓冲区溢出, 推荐使用fgets函数;
2. puts函数:
    - 函数原型: int puts(const char *s);
3. fgets函数:
    - 函数原型: char *fgets(char s[restrict .size], int size, FILE *restrict stream);
    - 缺陷: 只能保证缓冲区不溢出, 但是不能保证可以完整读取到一行;
4. getline函数
    - 不是c标准库函数, 是gnu-libc库函数;
    - 函数原型: ssize_t getline(char **restrict lineptr, size_t *restrict n, FILE *restrict stream);
